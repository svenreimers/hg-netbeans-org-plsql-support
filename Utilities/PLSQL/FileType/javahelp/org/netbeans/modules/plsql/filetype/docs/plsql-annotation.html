Annotations in PL/SQL Code

See Also

Editor Hints are available for some common mistakes in coding. Actions to correct the mistakes are provided with the hint where possible. User can avoid the annotation by adding the ignore marker if the warning/error is not applicable.

Annotations are available for the following categories.

  • Constants in package specification
  • Tags used for error/warning/info messages
  • Checks on function return
  • Checks on column comments
  • Checks on methods
  • Symbol '&' in comment blocks
  • COMMIT, ROLLBACK & SAVEPOINT statements in code
  • Common typing mistakes in cursor where clause
  • Checks on if condition
  • Checks on tables/views used in statements
